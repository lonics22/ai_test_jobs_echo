# 测试开发学习路线图（7 维进阶版）

> 目标：20k+ AI 测试工程师  
> 现状：7 年经验 / 1 年空窗 / AI 项目有框架但没深入  
> 核心策略：**以面试通过为终点，反向设计学习路径**

---

## 学习总纲

```
面试官想验证的 = 你能做什么 ≠ 你知道什么

所以学习的优先级是：
能说清楚的项目经验 > 能写出来的代码能力 > 理解的概念知识
```

**每个维度的学习分 3 层：**

| 层级 | 目标 | 验收标准 |
|------|------|---------|
| L1 理解 | 能用大白话解释概念 | 30 秒说清楚"这是什么" |
| L2 应用 | 能结合项目举例 | 60 秒说出你在项目中怎么用的 |
| L3 深入 | 能扛住追问和深挖 | 说出优缺点、改进方向、踩过的坑 |

---

## 维度一：测试基础

### 重要性：面试必考，但权重低（10%）
面试官不会在这块卡你，但你答得不好会减分。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | 软件开发生命周期、测试流程（需求→用例→执行→报告） | 看一遍，30 秒复述 | 1h |
| | 测试分类：功能/性能/安全/兼容/易用 | 背分类，各举一个项目中的例子 | 0.5h |
| | 测试策略：冒烟/回归/探索性/随机测试 | 说出每种策略在什么阶段用 | 0.5h |
| **L2 应用** | 能结合 Gate.io 项目说测试流程 | STAR 故事准备 | 1h |
| | 能说出你负责的模块用了什么测试策略 | 结合余币宝/Odoo | 1h |
| **L3 深入** | 缺陷分析：如何定位前端/后端/DB 问题 | 说一个线上 Bug 的排查链路 | 1h |
| | 质量度量：覆盖率、缺陷密度、逃逸率 | 说出你在项目中怎么衡量质量 | 1h |

### 🎯 面试高频问题
- "一个完整的测试流程是什么样的？"
- "你怎么定义'测够了'？"
- "功能测试、接口测试、UI 测试各自覆盖什么？"

## 维度二：用例设计

### 重要性：P0 必考（15%）
面试官一定会让你举例说明你怎么设计用例。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | 等价类划分 + 边界值分析 | 用"申购金额"举例说明 | 1h |
| | 场景法（基本流 + 备选流） | 用"登录→申购→成功/失败"举例 | 1h |
| | 正交实验法、因果图、判定表 | 知道什么场景用就行 | 1h |
| **L2 应用** | 结合余币宝申购功能，说出等价类 + 边界值 + 场景法的具体用例 | 40 条用例，覆盖核心场景 | 2h |
| | 结合 Odoo 采购审批流，说明场景法的备选流怎么识别 | 从历史 Bug 和业务规则中提取 | 2h |
| **L3 深入** | 如何保证用例的覆盖率？ | 用需求追踪矩阵(RTM)说明 | 1h |
| | 用例评审怎么做？评审标准是什么？ | 4 态流转：新建→评审→通过→驳回 | 1h |
| | AI 生成用例你怎么评估质量？ | 覆盖率、冗余度、有效性三维评估 | 1h |

### 🎯 面试高频问题
- "举例说明你怎么用等价类划分设计用例"
- "场景法你是怎么识别备选流的？"
- "你怎么判断用例写得够不够好？"

## 维度三：项目管理

### 重要性：中频，但决定了你是不是"只会干活的类型"（10%）

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | 敏捷开发流程（Scrum）、测试左移、质量门禁 | 30 秒解释 | 1h |
| | 风险评估：按资金 > 主流程 > 异常 > 辅助 定优先级 | 说一个你评估风险的案例 | 1h |
| **L2 应用** | 如何把控测试进度？项目延期了怎么办？ | PREP 准备 | 1h |
| | 跟开发意见不合怎么处理？ | 用数据说话的案例 | 1h |
| **L3 深入** | 如何推动团队重视质量？ | 量化数据 + 自动化门禁 | 1h |
| | 怎么定义质量指标并落地？ | Bug 率、自动化通过率、逃逸率 | 1h |

### 🎯 面试高频问题
- "给你一个新系统，你怎么制定测试策略？"（P0）
- "跟开发意见不合你怎么办？"（P0）
- "项目延期了你怎么办？"

## 维度四：UI 自动化

### 重要性：P1 高频（15%）。面试官想确认你是真做过还是只写过 demo。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | Selenium vs Playwright 区别 | 对比表格，各说优缺点 | 2h |
| | PO 模式（Page Object Model） | 画分层图：页面层/业务层/用例层 | 1h |
| | 元素定位策略（ID/XPath/CSS/文本） | 每种定位方式 1 个例子 | 1h |
| **L2 应用** | 搭建过 pytest + Selenium + Allure 框架 | 准备好分层架构说明 + 数据 | 3h |
| | 遇到过哪些稳定性问题？怎么解决的？ | 3 个问题：动态ID/数据污染/驱动兼容 | 2h |
| | UI 自动化维护成本怎么控制？ | 分层解耦 + 数据驱动 + 选型克制 | 1h |
| **L3 深入** | Selenium Grid 分布式执行 | 并发架构怎么设计 | 1h |
| | 失败用例自动重试机制 | pytest-rerunfailures 使用 | 1h |
| | AI Agent vs 传统 Selenium 的区别 | 各自优缺点、适用场景 | 1h |

### 🎯 面试高频问题
- "请介绍你搭建的自动化测试框架"（P0）
- "遇到过哪些稳定性问题？怎么解决的？"（P0）
- "接口自动化和 UI 自动化你怎么选型？"

## 维度五：接口自动化

### 重要性：P0 必考（15%）。这是你简历里的强项。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | HTTP 协议基础（GET/POST/PUT/DELETE、状态码、Header） | 30 秒解释 | 1h |
| | RESTful API 设计规范 | 知道 URL 结构、资源命名 | 1h |
| | pytest fixture 机制（scope、autouse、yield） | 结合项目举例 | 1h |
| **L2 应用** | 搭建 pytest + requests + Allure 框架 | 四层架构：配置/数据/业务/用例 | 3h |
| | 数据驱动测试（Excel/JSON/YAML） | 用例和代码解耦 | 1h |
| | 接口关联（token 传递、参数依赖） | fixture scope='session' 实现 | 1h |
| | Allure 报告配置（失败截图、请求日志） | 开发直接用报告定位 | 1h |
| **L3 深入** | 怎么保证接口自动化稳定可靠？ | 独立造数 + teardown 清理 + 重试 | 1h |
| | Mock 服务怎么搭建？什么场景需要？ | 第三方接口依赖时用 | 1h |
| | CI/CD 集成：Jenkins 定时执行 + 通知 | 企业微信通知、失败当晚修 | 1h |

### 🎯 面试高频问题
- "你的接口自动化框架怎么设计的？"（P0）
- "用例之间数据怎么隔离？"
- "接口自动化覆盖率多少？效果怎么样？"

## 维度六：性能测试

### 重要性：P1 中高频（10%）。至少能说出 JMeter 全流程。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | 性能指标：TPS、QPS、响应时间(P50/P95/P99)、并发数 | 每个指标 1 句话解释 | 1h |
| | JMeter 常用元件：线程组/取样器/监听器/断言 | 画一个完整的测试计划结构 | 2h |
| **L2 应用** | 怎么制定性能测试方案？ | 场景 → 指标 → 工具 → 数据 | 2h |
| | 全链路压测怎么做的？ | 核心接口 + 数据隔离 + 监控 | 2h |
| | 怎么定位性能瓶颈？ | DB/中间件/JVM 分层排查 | 2h |
| **L3 深入** | 数据库连接池优化、慢 SQL 分析 | 结合项目举例 | 2h |
| | 性能基线管理、容量规划 | 怎么判断系统能不能上线 | 1h |

### 🎯 面试高频问题
- "JMeter 常用的元件有哪些？"
- "性能测试流程是什么样的？"
- "发现性能瓶颈你怎么定位？"

## 维度七：AI 测试增强

### 重要性：**拉开差距的关键（25%）**。这是你和其他候选人最大的区别。

| 层级 | 内容 | 学习方式 | 时间 |
|------|------|---------|------|
| **L1 理解** | LLM 基础知识：预训练/微调/Token/幻觉/上下文窗口 | 1 句话说清楚 | 2h |
| | AI 在测试中的 3 个应用场景：生成用例/AI Agent/缺陷分析 | 每个场景 30 秒举例 | 1h |
| | AI 测试 vs 传统测试的差异 | 确定性 vs 概率性输出 | 1h |
| **L2 应用** | **AI 生成测试用例**：R.O.L.E.S.T 提示词框架 | 结合 TestHub 项目说明 | 3h |
| | **AI Agent（Browser-Use）**：架构 + 任务拆解 + 稳定性保障 | 做好被追问的准备 | 4h |
| | **AI 接口导入**：三阶段管线 + 参数分类策略 | 说明混合策略 | 2h |
| | **LVM 视觉模型**：需求文档图片处理 | 什么时候用、怎么用的 | 1h |
| **L3 深入** | **如何测试 AI Agent**：6 层评估体系 | 成功率/鲁棒性/安全/成本/一致性 | 3h |
| | **大模型幻觉检测**：5 层防御 | RAG + 管线 + 确定性校验 + Schema + 置信度 | 2h |
| | **Prompt 注入攻击测试**：安全围栏 | 攻击向量 + 防御策略 | 2h |
| | **MCP 协议**：让 AI 调工具的标准协议 | 结合 Skill 封装经验 | 2h |

### 🎯 面试高频问题

- "你在项目里怎么用 AI 的？"（追问到细节）

## 学习时间规划

### 今晚到明天面试前（紧急冲刺）

```
时间有限，只做能立刻提分的事：

1. 背熟自我介绍（1+3+1 结构）— 30min
2. 准备 3 个 STAR 故事（余币宝/Odoo/线上Bug）— 1h
3. 读 AI 项目 5 个核心问题的 PREP 答案 — 1h
4. 复习 JMeter 常用元件 + Python 装饰器/继承 — 30min
5. 准备 3 个反问 — 15min
```

### 第一周（脱敏 + 收集面试题）

```
目标：面 5 家公司，不管过不过

每天 2 小时：
- 第 1 天：完善 3 个 STAR 故事 + 自我介绍
- 第 2 天：准备 AI 项目回答（生成用例 + AI Agent）
- 第 3 天：接口自动化框架 + UI 自动化框架 回答
- 第 4 天：性能测试 + 用例设计
- 第 5 天：项目管理和行为面试题
- 周末：复盘本周面试，收集不会的题，补漏
```

### 第一个月（系统提升）

```
第 1 周：测试基础 + 用例设计（已掌握，只需要梳理成答案）
第 2 周：接口自动化框架体系（你的强项，重点打磨细节和数据）
第 3 周：UI 自动化 + 性能测试（补短板，重点解决说不清楚的问题）
第 4 周：AI 测试增强（拉开差距的关键）
```

### 第三个月（达到 20k+ 水平）

```
熟练回答简历上所有技术点
能独立完成 vibe coding 演示
能扛住 AI Agent 相关的 6 层追问
```

## 每日学习法

### 30 分钟学习法（适合你现在的状态）

```
00:00-05:00  读一个概念（如 pytest fixture）
05:00-10:00  用自己的话复述（录音）
10:00-15:00  想一个项目中的例子
15:00-20:00  说出来（录音）
20:00-25:00  听回放，改到流畅
25:00-30:00  写下来，存到面试答案库
```

关键：**不要连续读超过 10 分钟。10 分钟以后必须输出。**

## 最后

**别把所有维度都学完再去面试。** 学完 L1 + L2 的任意 3 个维度，你就已经能应付大多数面试了。

面试是**跑着学**的——面一场，补一块，越面越强。

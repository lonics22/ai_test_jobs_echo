### 🟡 第二周：Web Coding + 测试设计

面试中考"判断JSON格式"，但面试官真正考的是——**作为测试人员，怎么定义被测对象的边界**。

#### 黄金测试用例集（JSON 判断）

**有效等价类**：
- 合法 JSON 对象：`{"name":"test"}`
- 合法 JSON 数组：`[1,2,3]`
- 合法 JSON 字符串：`"hello"`
- 合法 JSON 数字：`42`、`3.14`、`-1`、`1e10`
- 合法 JSON 布尔：`true`、`false`
- 合法 JSON null：`null`
- 嵌套结构：`{"a":{"b":["x",true,null]}}`
- 空对象：`{}`
- 空数组：`[]`
- 空白字符处理：`  {"a":1}  `

**无效等价类**：
- 截断 JSON：`{"a":`
- 多余逗号：`{"a":1,}`
- 单引号代替双引号：`{'a':1}`
- 非 JSON 文本：`hello world`
- JavaScript undefined：`undefined`
- 函数表达式：`function(){}`
- 未闭合引号：`{"a":"hello}`
- 多重根元素：`{"a":1}{"b":2}`

#### 方法论：黄金测试用例集

```
种子集（人工定义，20-30条边界+等价类）
  → 基于种子生成回归集（AI辅助扩量到100+条）
    → 自动化验证 pipeline
      → 每次发布回归执行
```

**行动项**：针对 JSON 判断题目，写出完整 20+ 条测试用例并验证

### 🟡 第二周：Python 基础补漏

#### 装饰器 vs 继承 满分回答模板

**相同点**：都在不修改原始代码的前提下扩展功能的手段

**不同点**：
1. **粒度**：装饰器作用于函数/方法级别；继承作用于类级别
2. **维度**：装饰器是横向切面（AOP），解决横切关注点（日志/鉴权/重试）；继承是纵向层次（OO），解决 is-a 关系和多态
3. **组合方式**：装饰器是运行时组合；继承是编译时（静态语言）确定的层次结构
4. **测试视角**：装饰器适合统一注入通用行为（如重试、计时）；继承适合测试用例的层次化设计（如 BaseTestCase → ApiTestCase/UITestCase）

**行动项**：复习 Python 核心概念——装饰器、上下文管理器、元类、GIL、协程

## 六、核心教训

> **不是能力不够，是缺少"把做过的事情系统化表达"的训练。**

面试官其实一直在提示——问"黄金测试用例集"就是在说：你的测试集不应该全是AI生成的，要有一个人工定义的种子集做锚点。这个思路应该主动说出来，而不是等人提示。

### 关键转变：从执行者到架构师

| 现状（执行者） | 目标（架构师） |
|--------------|--------------|
| "这个功能我用XX实现了" | "这个模块我设计了XX架构，解决了XX问题" |
| "这块我不清楚" | "虽然不是我的领域，但从架构角度看..." |
| "加了等待时间" | "我设计了一个分层检测体系：前置校验→执行校验→逻辑校验→重试" |
| "组长负责这块" | "我们的并发架构是..." |

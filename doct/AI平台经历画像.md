```

   → LVM预处理（截图→文字）

   → revise修改
   → 状态"待确认" → 人工确认入库

③ 开发阶段 →  CI门禁
   开发提交代码
   → Jenkins自动触发
   → 接口回归（800条/20分钟）
   → Allure报告
   → 质量门禁：通过率<95% → 企微告警

④ 上线前验证
   人工验证高风险逻辑（资金安全/业务规则）

   → Playwright跑跨浏览器兼容
   → 结果确认 → 允许发布

⑤ 上线后
   线上异常自动上报
   → AI分析补充用例库
   → 失败案例反哺Prompt模板
```

## 🧩 我负责的 AI 模块

### 1️⃣ AI生成测试用例

**时机：** 需求评审后，测试上传PRD触发
**数据流：** 文档上传 → 文本提取 → LVM预处理（截图→文字）→ writer生成（SSE流式）→ reviewer评审 → revise修改 → 待确认 → 人工审核入库
**技术点：** 自动续写（finish_reason=length，最多5次）、断点修复、去重（Prompt预防+相似度比对）、失败反哺（reverse example沉淀Prompt）
**效果：** 累计生成3000+条，可用率70%+（最优85%），编写效率4-5h→1.5-2h

### 2️⃣ AI Agent UI自动化

**时机：** 版本上线前，测试手动触发
**不纳入CI原因：** 执行不稳定（85%）、Token成本高、时间长（40min）
**方案：** browser-use+LangChain，Agent主循环（DOM→LLM决策→执行→验证，最多100步）
**增强：** 11处monkey-patch适配国产模型 + 3层幽灵点击校验 + 超时重试 + 进度推送
**策略：** Selenium 70%做回归 + AI Agent 30%做复杂E2E

### 3️⃣ AI失败根因分析

**时机：** CI自动化跑完后自动触发
**方案：** 采集失败上下文（日志+报文+截图+console）→ LLM分类→ 代码报错打回开发/环境波动自动重跑/数据问题更新数据
**效果：** 定位时间15min→3min，准确率50%→70%

### 4️⃣ AI配置中心（AIModelConfig）

8种角色独立配置：writer/reviewer/vision/data_generator/field_classify/browser_use_text/browser_use_vision/api_import
每个角色可独立切换：provider/api_key/base_url/model/temperature
公司统一采购DeepSeek+智谱API Key

## 💡 设计哲学

1. **LLM做生成和分类，不做决策和控制** → AI出初稿，人工确认兜底
2. **AI Agent只做探索性测试，不做高频回归** → 不纳入CI，只在上线前跑
3. **三个引擎互补** → Selenium 70% / AI Agent 30% / Playwright 补充
4. **失败反哺闭环** → 评审不通过→沉淀为Prompt reverse example→迭代
5. **Prompt当代码管** → 版本控制、灰度验证、回滚机制

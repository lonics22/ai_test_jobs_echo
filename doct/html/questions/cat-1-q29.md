当前改进思路主要有两个方向：

**第一，多 Agent 协作。** 现在单 Agent 能力有限，质量天花板明显。后续想做多 Agent 协作——一个 Agent 理解需求、一个生成用例、一个评审、一个执行、一个分析结果。它们互相对话和交叉验证，评审 Agent 发现问题直接打回修改，减少人工介入。

**第二，AI 生产分析——从被动工具变成主动发现问题。** 目前 AI 更多是等人工指令的被动角色。后续想接入生产环境运行日志，让 AI 主动分析线上流量特征：监控接口响应时间趋势、识别高频报错的参数组合、分析调用链数据。AI 不再等人说"帮我查一下这个日志"，而是主动推送——"接口 A 的 TP99 最近三天上升 50%，建议排查"或"接口 B 的参数组合 X 在线上频繁报错，测试未覆盖，建议补充用例"。从人用 AI 的工具，转变成 AI 主动产出分析、推动质量改进的驱动力。

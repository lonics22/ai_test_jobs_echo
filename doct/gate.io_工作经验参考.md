## Gate.io — AI 一体化测试平台 / 测试开发（2025.10-2026.06）

负责公司 AI+测试一体化平台的架构设计与落地，后端基于 Django 4.2 + DRF + Celery，前端使用 Vue 3 + Element Plus + Vite，AI 层接入 DeepSeek、通义千问等大模型，将智能能力融入需求分析、用例设计、接口测试、缺陷分析等测试全流程。同时承担交易所核心交易链路（永续合约、余币宝理财）的质量保障工作，覆盖功能、接口、性能、资金安全多个维度。

- 自研 AI 需求解析引擎，支持 PDF/Word/TXT 多格式文档自动解析与业务实体抽取，接入 LVM 视觉大模型识别文档中的图片和流程图，需求分析到用例输出的周期从按天缩短到分钟级，覆盖 10+ 个项目
- 搭建提示词工程体系，定制 5 套 prompt 模板分别用于用例生成、评审加固、测试数据构造等场景，AI 生成用例有效率达 85%+，累计沉淀可复用用例 3000+
- 开发 AI 接口文档解析模块，通过 LLM 混合参数分类算法自动识别 Swagger/OpenAPI/Postman/HAR 四种格式的接口定义，接口导入效率提升 70%，累计处理 200+ 份 API 文档
- 基于 Python + requests + pytest + allure 搭建接口自动化体系，覆盖合约开多/开空、撤单、仓位管理、资金费率结算、强平触发、阶梯保证金切换及理财申购/赎回等核心接口，接入 Jenkins 实现每日持续回归，回归耗时减少 30%
- 设计数据对比与异常检测机制，基于交易日志和用户行为数据自动校验资金费率、清算结果、强平价格等关键数据的准确性，保障用户资产安全
- 打通 AI 能力全流程链路，NLP 缺陷智能分析命中率达 80%+，接入 CI/CD 质量门禁后线上漏测率降低 35%
- 开展 LLM 应用安全专项测试，涵盖 Prompt 注入、隐私合规、对抗攻击、幻觉输出、敏感信息泄露等方向，沉淀评测集作为每次迭代的质量基线

## 面试高频问题方向

1. **合约交易的核心测试要点？**
   - 保证金计算验证、强平触发条件、资金费率计算准确性
   - 订单状态机转换（已成交/部分成交/撤单/失效）
   - 高并发下的撮合引擎性能与数据一致性

2. **怎么保证数字资产资金安全？**
   - 钱包余额与理财账户对账、收益计算精度验证
   - 异常补偿机制测试、风险拦截规则验证
   - 幂等性校验（防重复申购/赎回）

3. **AI 在测试中怎么落地？**
   - 测试用例 AI 生成提效
   - LLM 应用本身的专项安全测试
   - 测试报告自动化

4. **交易所性能瓶颈在哪？**
   - 撮合引擎并发处理能力
   - 数据库连接池在高频下单场景下的表现
   - 清算/资金费率结算的定时任务压力

# Interview Brief Evaluation Rubric

判断备书是否真的解决问题。

## 必须达标

| 指标 | 达标标准 |
|-|-|
| JD 贴合度 | 每个能力维度都能回到 JD 职责或要求 |
| 来源标注 | 每道题标注真题参考 / 相似题迁移 / JD 推演题 |
| 直接可用性 | 至少 5 道 P0/P1 核心题有 60-120 秒可直接朗读的参考答案 |
| 可训练性 | 至少 5 道核心题有答案骨架、参考答案和追问链路 |
| 数据密度 | 每道直接答案至少包含 2 个业务/产品指标，真实数据缺失时给占位与示例口径 |
| 案例连续性 | 连续追问围绕同一个项目故事深挖，指标和业务背景前后一致 |
| 个人化 | 明确候选人还缺哪些项目、指标或案例 |
| 风险控制 | 不把推演题包装成真题 |

## 好答案标准

- 能解释面试官为什么问这道题
- 能给出可替换个人经历的位置
- 能提供一段用户马上可以开口练的参考答案
- 有指标口径，不只说"提升用户体验"
- 有业务细节，不空谈方法论
- 有追问预判
- 能区分一面、二面、HR 面关注点

## 常见失败

- 只列题目，不解释能力点
- 只有框架，没有可直接练习的答案文本
- 答案像标准作文，用户无法替换成自己的经历
- 参考答案过于完美或虚构，没有标出个人素材占位
- 参考答案没有数据，只说"效果变好""体验提升"
- 多个问题里案例故事和指标互相打架
- 题目看似很多，但没有优先级
- 没有追问，无法训练真实面试压力
- 混淆真题和推演题
